# 辩论评审模式

多智能体辩论评审，产出高质量决策。

## 核心理念

> 真理越辩越明

一个方案好不好，让多个智能体从不同角度质疑、辩护，最终由裁判智能体综合裁决。

## 使用方法

```bash
# 启动辩论
spawn_agent({ 
  skill: "debate-review",
  task: "评审游戏定位方案：Roguelike卡牌游戏，目标用户是25-35岁男性..."
})
```

## 辩论流程

```
第1轮：提案
┌─────────────────────────────────────────────┐
│ 策划CODEX：提出方案                          │
│ - 市场分析                                   │
│ - 游戏定位                                   │
│ - 核心玩法                                   │
└─────────────────────────────────────────────┘
                    ↓
第2轮：质疑
┌─────────────────────────────────────────────┐
│ 质疑CODEX：挑毛病                            │
│ - 市场数据是否过时？                         │
│ - 竞品分析是否全面？                         │
│ - 目标用户是否太窄？                         │
│ - 差异化是否足够？                           │
└─────────────────────────────────────────────┘
                    ↓
第3轮：辩护
┌─────────────────────────────────────────────┐
│ 策划CODEX：回应质疑                          │
│ - 补充数据来源                               │
│ - 解释选择理由                               │
│ - 承认不足并提出改进                         │
└─────────────────────────────────────────────┘
                    ↓
第4轮：再质疑（可选）
┌─────────────────────────────────────────────┐
│ 质疑CODEX：追问                              │
│ - 改进是否足够？                             │
│ - 是否有更优方案？                           │
└─────────────────────────────────────────────┘
                    ↓
第N轮：裁决
┌─────────────────────────────────────────────┐
│ 裁判CODEX：综合评判                          │
│ - 双方论点总结                               │
│ - 关键争议点                                 │
│ - 最终决策                                   │
│ - 行动建议                                   │
└─────────────────────────────────────────────┘
```

## 角色设定

### 策划CODEX（正方）
```
你是策划智能体，负责提出和辩护方案。

职责：
- 提出完整方案
- 用数据和逻辑支撑观点
- 回应质疑
- 承认合理批评并改进

态度：自信但不固执，开放但有底线
```

### 质疑CODEX（反方）
```
你是质疑智能体，负责挑毛病和找漏洞。

职责：
- 从多个角度质疑方案
- 挑战假设和前提
- 提出替代方案
- 追问不清晰的地方

态度：尖锐但不刻薄，严谨但不钻牛角尖
```

### 裁判CODEX（主控）
```
你是裁判智能体，负责主持辩论和最终裁决。

职责：
- 主持辩论流程
- 识别关键争议点
- 综合双方观点
- 给出最终决策

态度：公正客观，追求最优解而非妥协
```

## 评审维度

| 维度 | 质疑点 |
|------|--------|
| **市场** | 数据来源？样本量？时效性？竞争格局？ |
| **用户** | 用户画像是否准确？需求是否真实？ |
| **产品** | 核心玩法是否有趣？差异化是否足够？ |
| **技术** | 是否可实现？风险点？ |
| **商业** | 成本收益？定价策略？回本周期？ |
| **风险** | 政策风险？市场风险？技术风险？ |

## 实现代码

```typescript
async function runDebateReview(topic: string, rounds: number = 3) {
  const mainAgent = getActiveAgent();
  
  // 创建正方
  const proponentId = await spawnAgent(mainAgent, {
    role: 'proponent',
    systemPrompt: PROONENT_PROMPT
  });
  
  // 创建反方
  const opponentId = await spawnAgent(mainAgent, {
    role: 'opponent', 
    systemPrompt: OPPONENT_PROMPT
  });
  
  // 辩论循环
  let proposal = await executeAgent(proponentId, `提出方案：${topic}`);
  
  for (let i = 0; i < rounds; i++) {
    // 质疑
    const challenge = await executeAgent(opponentId, `质疑方案：${proposal}`);
    
    // 辩护
    const defense = await executeAgent(proponentId, `回应质疑：${challenge}`);
    
    proposal = defense;
  }
  
  // 裁判
  const verdict = await executeAgent(mainAgent, `
    作为裁判，综合以下辩论给出最终裁决：
    
    方案：${proposal}
    质疑历史：...
    
    输出最终报告。
  `);
  
  return verdict;
}
```

## 适用场景

| 场景 | 推荐轮数 |
|------|----------|
| 简单决策 | 1-2轮 |
| 方案评审 | 2-3轮 |
| 战略决策 | 3-5轮 |
| 高风险决策 | 5轮+ |

## 注意事项

1. **成本**：每轮辩论消耗 Token，控制轮数
2. **效率**：简单问题不需要辩论，直接决策
3. **质量**：反方质疑质量决定最终方案质量
4. **收敛**：避免无限辩论，设置最大轮数
